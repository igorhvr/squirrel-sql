Hibernate Plugin

To use the Hibernate Plugin requires to create at least one Hibernate configuration in SQuirreL's Global Preferences.




You start out with providing a name for the configuration. Than you add all the Jars of your Hibernate installation as well as all jars that are needed by your mapped classes and the mapped classes themselves to the additional Classpath. Normally this is simply the same Classpath as you use in your project.

Since there are several way of obtaining a Hibernate connection (SessionFactory) the Hibernate Plugin leaves it to the user to provide this connection. This means that the user has to give the Plugin a class that has a static method with the following signature:

public org.hibernate.impl.SessionFactoryImpl getSessionFactoryImpl()

The Plugin will obtain the Hibernate connection using this function. The dialog to tell the Plugin the name of this class offers to save an example implementation such a class. Note that this class must be inside your additional Classpath too.




Note: Never forget use the “Apply changes to this configuration” button when you are done.



When this configuration part is done you may connect to a configuration from inside a Session window.
